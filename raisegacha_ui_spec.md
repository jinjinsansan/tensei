# 来世ガチャ UI リデザイン仕様書

> **サイト名**: 来世ガチャ ～もしも生まれ変わったら～
> **世界観**: 輪廻の書庫 — 天空の図書館
> **対象プロジェクト**: tensei-gacha（尊師ガチャ V4 からの移植）
> **ステータス**: Claude Code 実装指示用

---

## 0. はじめに — この仕様書の目的

尊師ガチャ V4 のロジック・API・DB 設計はそのまま流用し、**サイト全体の見た目・世界観・テキスト・演出**を「輪廻の書庫」テーマに置き換える。本仕様書はフロントエンド実装者（Claude Code）が迷わず作業できるよう、画面単位で具体的な指示を記載している。

**原則:**
- ガチャエンジン（RTP / どんでん / 追撃 / カード配布）のロジックは一切変更しない
- API のリクエスト・レスポンス構造は変更しない
- 変更するのは UI コンポーネント、CSS / Tailwind クラス、テキスト、画像アセットのみ

## 1. 世界観コンセプト

### 1.1 設定

> ここは「輪廻の書庫」。
> 雲の上に浮かぶ、果てしなく広い図書館。
>
> この書庫には、すべての人間の「人生の本」が収められている。
> 過去の人生も、未来の人生も、すべてがここに。
>
> あなたの目の前に、一冊の光る本が現れた。
> その本を開けば、新しい人生が始まる。
>
> コンビニ店員の人生？ 世界的アーティストの人生？
> それとも——
>
> さあ、ページをめくって。

- 天空の図書館、無限の本棚、アンティーク、魔法の本
- ランプの光、金色の塵、革装丁、箔押し

- **温かみのある神秘感** — ダークファンタジーではなくジブリ的な優しい不思議さ

## 5. アニメーション & エフェクト

### 5.1 共通パーティクル: 金色の塵

全ページの背景に、微かに金色の塵が漂うエフェクトを追加:

```css
@keyframes float-particle {
  0%   { transform: translateY(100vh) translateX(0); opacity: 0; }
  10%  { opacity: 0.6; }
  90%  { opacity: 0.6; }
  100% { transform: translateY(-10vh) translateX(20px); opacity: 0; }
}

.particle {
  position: fixed;
  width: 3px;
  height: 3px;
  background: #C9A84C;
  border-radius: 50%;
  pointer-events: none;
  animation: float-particle 8s linear infinite;
}
```

**実装:** ルートレイアウト (`layout.tsx`) に `<BackgroundParticles />` コンポーネントを追加。10〜15個の粒子を `position: fixed` で配置し、それぞれ異なる `animation-delay` と `left` を設定。**パフォーマンスに注意:** `will-change: transform` を使用し、ガチャ再生中は非表示にする。

### 5.2 ページ遷移

```
フェードイン: opacity 0→1, 0.3s ease
スライドイン: translateY(20px) → 0, 0.3s ease（下から上）
```

### 5.3 カード出現アニメーション

結果画面でカードが表示される際:

```
1. 背景が暗転（0.3s）
2. 本が画面中央に出現（scale 0→1, 0.5s, ease-out）
3. 本が光る（box-shadow パルス, 0.5s）
4. 本が開く（rotateY, 0.5s）→ カード面が見える
5. レア度に応じたパーティクル爆発（0.3s）
```

## 7. レスポンシブ & アクセシビリティ

### 7.1 レスポンシブ対応

```
モバイル（< 640px）: 1カラム、フルワイド
タブレット（640-1024px）: 中央寄せ max-w-lg
デスクトップ（> 1024px）: 中央寄せ max-w-md（9:16 比率を維持）
```

ガチャプレイヤーは全画面 Portal なので、レスポンシブ不要（常に画面いっぱい）。

### 7.2 ダークモード

デフォルトでダーク系のため、**ライトモードは実装しない**（世界観と合わない）。`<html>` に `class="dark"` を固定。

### 7.3 アクセシビリティ

- 全ボタンに `aria-label` 付与
- 色のコントラスト比: テキスト (#F5E6D3) on 背景 (#2C1810) = 8.5:1（AAA 適合）
- フォーカスリング: `outline: 2px solid #C9A84C`
- 動きの低減: `prefers-reduced-motion` で パーティクル / アニメーション無効化

## 9. 注意事項

1. **ロジックに触らない**: ガチャエンジン (`generator.ts`)、API ルート、DB スキーマは一切変更しない
2. **段階的に進める**: Phase 1 → 2 → 3 → 4 の順番を守る。基盤が固まらないまま画面を作ると手戻りが発生する
3. **テロップ画像は仮置きOK**: 新画像の制作には時間がかかるため、テキストで仮表示し後から差替える方針でOK
4. **パフォーマンス**: パーティクルや背景エフェクトは `will-change` / `transform` を使い、GPU アクセラレーションを活用。ガチャ再生中は不要なエフェクトを停止する
5. **Tailwind との統合**: カラーパレットは `tailwind.config.ts` の `extend.colors` に追加し、`bg-primary`, `text-accent` 等で使えるようにする
